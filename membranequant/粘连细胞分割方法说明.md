# 粘连细胞分割方法说明

## 概述

当细胞生长密集或存在粘连时，传统的Otsu阈值分割会将多个细胞识别为一个连通域，导致分割失败。本工具提供了多种经典图像处理方法来解决这个问题，这些方法来自ImageJ、文献和OpenCV等成熟工具。

## 可用方法

### 1. Otsu 阈值法（默认）
**适用场景**：细胞分离良好，无明显粘连

**原理**：
- 自动计算最优阈值将前景和背景分离
- 使用形态学操作（closing）清理噪声
- 填充内部空洞

**优点**：
- 速度快
- 无需参数调整
- 结果稳定

**缺点**：
- 对粘连细胞无能为力
- 会将粘连细胞识别为一个大的连通域

---

### 2. 距离变换+分水岭 (watershed_distance)
**适用场景**：圆形或椭圆形的粘连细胞

**原理**：
1. 二值化得到前景区域
2. 计算距离变换：每个前景像素到边界的距离
3. 在距离图的局部最大值处放置种子（这些是细胞中心）
4. 从种子点开始，沿着距离图的"山谷"生长，直到相邻细胞的边界相遇

**参考**：
- ImageJ的Watershed插件核心算法
- OpenCV官方教程：Watershed segmentation
- 中文文献：基于距离变换的分水岭算法分割粘连物体

**优点**：
- 对圆形/椭圆形细胞效果很好
- 是ImageJ中最常用的粘连分离方法
- 参数少，鲁棒性好

**缺点**：
- 对形状不规则的细胞效果一般
- 可能产生过度分割

**何时使用**：
- 细胞大致呈圆形
- 粘连点较窄
- **这是处理粘连的首选方法**

---

### 3. 梯度+分水岭 (watershed_gradient)
**适用场景**：细胞边界清晰（荧光强度在边界处有明显下降）

**原理**：
1. 计算图像梯度（Sobel算子），得到边界强度图
2. 通过形态学opening操作得到确定的细胞内部区域作为种子
3. 在梯度图上进行分水岭分割
4. 算法会优先在梯度高（边界强）的地方划分边界

**参考**：
- ImageJ Morphological Segmentation插件
- Marker-controlled watershed for cell segmentation

**优点**：
- 利用了边界信息，对边界清晰的图像效果好
- 能够较好地贴合真实细胞边界

**缺点**：
- 对噪声敏感
- 如果细胞内部也有强梯度，可能导致错误分割

**何时使用**：
- EGFP荧光在细胞边界处明显减弱
- 图像信噪比较高

---

### 4. H-minima+分水岭 (hminima_watershed)
**适用场景**：密集培养的细胞，容易产生过度分割

**原理**：
1. 计算距离变换
2. 应用H-minima变换：抑制深度小于h的局部极小值
3. 保留的极小值作为细胞中心的marker
4. 进行分水岭分割

**H-minima的作用**：
- 普通分水岭可能把一个细胞的内部凹陷识别为多个细胞
- H-minima会"填平"浅的凹陷，只保留深的凹陷（真正的细胞中心）
- 有效减少过度分割

**参考**：
- "Iterative h-minima-based marker-controlled watershed for cell nucleus segmentation" (2016)
- "Marker-controlled watershed with deep edge emphasis and optimized H-minima transform" (2022)
- 这是文献中针对密集细胞核/细胞分割的常用方法

**优点**：
- 能有效抑制过度分割
- 适合密集培养的细胞
- 文献验证的可靠方法

**缺点**：
- h值的选择会影响结果（本工具自动设为距离最大值的30%）
- 可能在某些情况下漏检细胞

**何时使用**：
- 细胞非常密集
- 使用距离变换分水岭时发现一个细胞被分成多个
- **推荐作为粘连分割的第二选择**

---

### 5. 形态学开运算 (morphological_opening)
**适用场景**：轻度粘连，细胞间仅有细窄连接

**原理**：
1. 二值化
2. 开运算 = 腐蚀(erosion) + 膨胀(dilation)
   - 腐蚀：收缩前景，可以断开细窄连接
   - 膨胀：恢复细胞大小
3. 最终得到分离的细胞

**参考**：
- ImageJ: Process > Binary > Watershed
- 形态学操作是图像处理的经典方法

**优点**：
- 原理简单直观
- 速度快
- 对轻度粘连效果好

**缺点**：
- 只适合粘连点很窄的情况
- 无法处理大面积粘连
- 会改变细胞形状（略微变圆）

**何时使用**：
- 细胞主要是分离的，只有少数接触点
- 粘连区域很窄（< 5像素）

---

### 6. 距离+梯度双重markers (combined_markers)
**适用场景**：复杂场景，综合性能最优

**原理**：
1. 使用距离变换找到细胞中心作为markers（确保每个细胞有种子）
2. 计算梯度图（利用边界信息）
3. 结合距离信息和梯度信息构建"地形图"
4. 在这个综合地形图上进行分水岭分割

**优点**：
- 综合了细胞中心信息和边界信息
- 比单一方法更稳定
- 对各种类型的粘连都有一定效果

**缺点**：
- 计算稍慢
- 参数较多（虽然已自动调整）

**何时使用**：
- 不确定用哪种方法
- 其他方法效果都不理想
- **推荐作为"保底"方案**

---

### 7. Cellpose 深度学习 (cellpose)
**适用场景**：极其复杂的粘连，或形状非常不规则的细胞

**原理**：
- 使用深度神经网络识别细胞
- 网络在数万张细胞图像上训练过
- 能够学习复杂的细胞形态特征

**安装**：
```bash
pip install cellpose
```

**优点**：
- 效果通常是最好的
- 能处理各种复杂情况
- 对形状不规则的细胞也有效

**缺点**：
- 需要额外安装（约500MB）
- 首次运行需要下载模型
- 速度较慢（尤其是CPU）
- 需要一定的GPU和内存

**何时使用**：
- 所有传统方法都失败
- 愿意等待更长时间
- 有GPU加速更佳

## 使用建议

### 针对粘连细胞的推荐流程：

```
1. 先试 Otsu 阈值法
   ↓ 如果有粘连
   
2. 试 距离变换+分水岭 (watershed_distance)
   ↓ 如果过度分割（一个细胞分成多个）
   
3. 试 H-minima+分水岭 (hminima_watershed)
   ↓ 如果还是不行
   
4. 试 距离+梯度双重markers (combined_markers)
   ↓ 如果仍不满意且有条件
   
5. 安装并试 Cellpose
```

### 参数说明

所有分水岭方法都会自动根据 `minimum_cell_area`（最小细胞面积）参数来估算合理的种子点间距，无需手动调整。

如果觉得分割出的细胞太多或太少，可以调整：
- **最小细胞面积** (`minimum_cell_area`)：增大会减少检测到的小细胞
- **最大细胞面积** (`maximum_cell_area`)：减小会排除粘连团
- **分水岭方法**：换一个方法试试

## 实际效果示例

查看生成的叠加图 (`overlays/` 文件夹) 来评估分割效果：
- **绿色轮廓**：细胞边界
- **红色区域**：膜环
- **蓝色区域**：胞质
- **数字标签**：细胞编号

好的分割应该：
1. 每个细胞有独立的编号
2. 边界贴合实际细胞
3. 粘连的细胞被正确分开
4. 没有过度分割（一个细胞分成多个）
