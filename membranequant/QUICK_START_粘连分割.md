# 粘连细胞分割 - 5分钟快速入门

## 🎯 你的问题：细胞粘在一起，Otsu分不开

## ✅ 解决方案：用watershed分水岭算法

## 🚀 3步开始

### 第1步：启动Web界面

双击运行：
```
run_webui.cmd
```

或在命令行：
```bash
cd D:\杂物\grok
python -m membranequant.webui
```

浏览器自动打开 `http://127.0.0.1:7860`

---

### 第2步：选择分割方法

在"**细胞分割方法**"下拉菜单中：

**如果细胞粘连，选这个** 👇

```
距离变换+分水岭（适合圆形粘连细胞）
```

这是ImageJ最经典的方法，对圆形/椭圆形细胞效果最好。

---

### 第3步：运行分析

1. 填写"输入：实验数据文件夹"
2. 点击"**预览配对**"确认图片正确识别
3. 点击"**开始分析**"

完成后查看 `overlays/` 文件夹的叠加图，检查分割效果。

## 🔄 效果不好？试试这些

### 问题：一个细胞被分成多个（过度分割）

**解决**：换成 **H-minima+分水岭**

```
H-minima+分水岭（适合密集粘连细胞）
```

这个方法会抑制浅的凹陷，减少过度分割。

---

### 问题：边界不准确

**解决**：试试 **距离+梯度双重markers**

```
距离+梯度双重markers（综合方法）
```

这是综合方法，稳定性最好。

---

### 问题：所有方法都不行

**解决**：安装Cellpose（深度学习）

```bash
pip install cellpose
```

然后选择：
```
Cellpose 深度学习（需额外安装）
```

**注意**：Cellpose比其他方法慢很多，首次运行需要下载模型。

## 🎨 如何判断分割好不好

打开 `Results/overlays/` 文件夹：

### ✅ 好的分割
- 每个细胞有独立的编号
- 粘连的细胞被正确分开
- 边界贴合实际细胞形态
- 颜色：绿=细胞边界，红=膜环，蓝=胞质

### ❌ 不好的分割
- 多个细胞粘成一团（编号相同）→ 换watershed方法
- 一个细胞分成多个（编号不同）→ 换hminima方法
- 边界乱七八糟 → 检查图像质量或试试combined_markers

## 💡 常见问题

### Q: 需要安装什么吗？
A: 不需要！所有新方法都用现有依赖。只有Cellpose需要额外安装。

### Q: 会不会很慢？
A: 不会。watershed方法和Otsu速度差不多，比Cellpose快很多。

### Q: 参数需要调吗？
A: 通常不需要。如果效果不好，先换方法，再考虑调参数。

### Q: 原来的数据需要重新分析吗？
A: 不需要，除非你想用新方法获得更好的结果。

### Q: 哪个方法最好？
A: 
- 粘连不严重 → 距离变换分水岭
- 粘连严重 → H-minima分水岭
- 不确定 → 距离+梯度双重markers（最稳定）

## 🎓 技术背景（可跳过）

### 为什么分水岭算法有效？

想象一下：
1. 细胞像山丘，边界是山谷
2. 算法在每个山丘顶部放一个"种子"（细胞中心）
3. 从种子开始"注水"，水慢慢扩散
4. 两个细胞的水相遇的地方就是边界

### 距离变换是什么？

计算每个像素到边界的距离：
- 细胞中心的距离最大（山丘顶部）
- 靠近边界的距离小（山坡）
- 边界的距离为0（山谷底）

### H-minima为什么能减少过度分割？

它会"填平"浅的山谷（细胞内的小凹陷），只保留深的山谷（真正的细胞分界）。

## 🔬 实战案例

### 场景1：正常培养的HeLa细胞
- **特点**：偶有接触，大部分分离
- **推荐**：Otsu（默认）
- **备选**：watershed_distance

### 场景2：密集培养的293T细胞
- **特点**：大量粘连，形状较圆
- **推荐**：watershed_distance
- **备选**：hminima_watershed

### 场景3：汇合前的原代细胞
- **特点**：粘连严重，形状不规则
- **推荐**：combined_markers
- **备选**：watershed_distance → cellpose

### 场景4：3D培养/类器官
- **特点**：极度复杂，重叠严重
- **推荐**：cellpose（需安装）
- **备选**：如果是2D投影，可以试试hminima_watershed
